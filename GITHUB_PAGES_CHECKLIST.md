# GitHub Pages publication checklist

Before publishing:

- Confirm that the `Images/` and `assets/` folders are present in the repository.
- Confirm that no passwords, API keys, private exports, chat archives, or unrelated personal files are tracked.
- In **Settings → Pages**, select the intended publishing branch/folder.
- Enable **Enforce HTTPS** as soon as GitHub makes the option available.
- Open `index.html`, `legal.html`, `privacy.html`, and `imprint.html` from the published site and test their links.
- Verify that `kurokunikami [at] gmail.com` is reachable.

This checklist applies to the static GitHub Pages release without a backend, accounts, or server-side uploads.
